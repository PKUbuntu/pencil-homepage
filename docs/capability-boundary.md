# .pen 文件解析能力边界

> 本文档定义了 .pen 文件格式中可解析和不可解析的内容范围

## ✅ 完全支持的特性

### 元素类型

| 元素类型 | 支持状态 | 说明 |
|----------|----------|------|
| `frame` | ✅ 完全支持 | 容器/矩形元素，支持所有属性 |
| `text` | ✅ 完全支持 | 文本元素，支持所有排版属性 |

### 布局属性

| 属性 | 支持状态 | 说明 |
|------|----------|------|
| `layout: "vertical"` | ✅ | 垂直 Flexbox 布局 |
| `layout: "horizontal"` | ✅ | 水平 Flexbox 布局 |
| `layout: "none"` | ✅ | 无自动布局 |
| `justifyContent` | ✅ | `"center"`, `"space_between"` |
| `alignItems` | ✅ | `"center"`, `"end"` |
| `flexWrap` | ✅ | `"wrap"` |
| `gap` | ✅ | 子元素间距 |
| `padding` | ✅ | 单值、双值、四值 |

### 尺寸系统

| 值类型 | 支持状态 | 说明 |
|--------|----------|------|
| 固定数值 | ✅ | 像素值（如 `1440`, `320`） |
| `"fill_container"` | ✅ | 填充父容器 |
| `"fit_content"` | ✅ | 自适应内容 |
| `"fit_content(N)"` | ✅ | 带最小值提示的自适应 |

### 样式属性

| 属性 | 支持状态 | 说明 |
|------|----------|------|
| `fill`（颜色值） | ✅ | Hex 颜色，如 `"#ffffff"` |
| `fill`（变量引用） | ✅ | 变量引用，如 `"$--background"` |
| `fill: "transparent"` | ✅ | 透明填充 |
| `stroke.fill` | ✅ | 描边颜色 |
| `stroke.thickness` | ✅ | 描边宽度 |
| `cornerRadius`（单值） | ✅ | 等值圆角 |
| `cornerRadius`（数组） | ✅ | 分角圆角 `[tl, tr, br, bl]` |
| `opacity` | ✅ | 不透明度 |

### 文本属性

| 属性 | 支持状态 | 说明 |
|------|----------|------|
| `content` | ✅ | 文本内容 |
| `fontFamily` | ✅ | 字体族 |
| `fontSize` | ✅ | 字体大小 |
| `fontWeight` | ✅ | 字重（`"bold"`, `"500"`, `"600"` 等） |
| `fontStyle` | ✅ | 字体样式（如 `"italic"`） |
| `textAlign` | ✅ | 文本对齐 |
| `lineHeight` | ✅ | 行高 |
| `textGrowth` | ✅ | 文本增长模式 |

### 主题系统

| 特性 | 支持状态 | 说明 |
|------|----------|------|
| 主题维度定义 | ✅ | `Mode`, `Base`, `Accent` |
| 主题条件变量值 | ✅ | 基于 `theme` 匹配的条件值 |
| Frame 级主题覆盖 | ✅ | 通过 `theme` 属性激活 |

### 变量系统

| 特性 | 支持状态 | 说明 |
|------|----------|------|
| 颜色变量定义 | ✅ | `type: "color"` |
| 变量引用 (`$--name`) | ✅ | 在 `fill`、`stroke.fill` 中引用 |
| 多值条件变量 | ✅ | 默认值 + 主题条件值 |

### 结构特性

| 特性 | 支持状态 | 说明 |
|------|----------|------|
| 嵌套子元素 | ✅ | 多层嵌套的 `children` 数组 |
| 元素 ID | ✅ | 唯一标识符 |
| 元素命名 | ✅ | 人类可读的 `name` 属性 |
| 画布坐标 | ✅ | 顶层 Frame 的 `x`, `y` |
| 裁剪 (`clip`) | ✅ | 内容溢出裁剪 |

## ⚠️ 部分支持的特性

| 特性 | 状态 | 说明 |
|------|------|------|
| 多主题维度 | ⚠️ | `Base` 和 `Accent` 维度已声明但当前文件未实际使用 |
| 非颜色变量 | ⚠️ | 变量系统仅发现 `type: "color"`，理论上可能支持其他类型但未见实例 |
| 直接颜色混合使用 | ⚠️ | 部分元素混用直接颜色值（`"#22c55e"`）和变量引用（`"$--accent"`），需注意解析优先级 |

## 解析建议

### 优先级 P0：核心解析能力

1. **JSON 解析**：标准 JSON，无需特殊处理
2. **Frame 元素**：递归解析 `children` 树
3. **Text 元素**：提取 `content` 和排版属性
4. **变量解析**：替换 `$--name` 引用为实际颜色值
5. **主题解析**：根据目标主题选择正确的变量值
6. **布局计算**：Flexbox 布局映射到 CSS

### 优先级 P1：增强能力

1. **尺寸计算**：处理 `fill_container`、`fit_content` 等动态尺寸
2. **圆角数组**：处理分角圆角
3. **描边样式**：应用描边到对应元素
4. **不透明度**：处理 `opacity` 属性

### 优先级 P2：可选能力

1. **多主题维度**：支持 `Base` 和 `Accent` 维度的变量切换
2. **非颜色变量**：预留其他类型变量的扩展能力
3. **图片元素**：未来可能出现 `image` 类型元素

---

## 总结

| 维度 | 评估 |
|------|------|
| 格式复杂度 | **低** — 纯 JSON，仅 2 种元素类型 |
| 解析可行性 | **高** — 结构清晰，属性有限且明确 |
| 主要挑战 | 变量解析 + 主题条件值计算 + 动态尺寸 |
| 最大限制 | 无图片、无组件引用、无矢量图形 |
| 适用场景 | 适合生成静态页面、组件代码、设计文档 |
