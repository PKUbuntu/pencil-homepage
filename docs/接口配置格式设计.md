# 接口配置格式设计

## 概述

本文档定义了 Pencil Homepage 项目的接口配置格式，支持声明式定义 API 接口，自动生成后端代码。配置采用 YAML 格式（优先），也可使用 JSON。

## 设计目标

- **声明式**：通过配置文件描述 API，无需手写样板代码
- **类型安全**：使用 Zod Schema 进行运行时类型验证
- **多协议支持**：同时支持 REST 和 GraphQL 接口
- **自动生成**：根据配置自动生成 tRPC Router 和 WunderGraph 配置
- **易于维护**：配置清晰易懂，便于团队协作

## 配置格式规范

### 根结构

```yaml
version: "1.0"           # 配置格式版本
apis: []                 # API 定义列表
types: {}                # 自定义类型定义
validation: {}           # 全局验证规则
```

### API 定义

每个 API 包含以下字段：

| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| `name` | string | ✅ | API 名称（驼峰命名） |
| `type` | string | ❌ | 接口类型：`rest` 或 `graphql`（默认：rest） |
| `method` | string | ❌ | HTTP 方法：`GET`/`POST`/`PUT`/`DELETE`/`PATCH`（默认：GET） |
| `path` | string | ✅ | API 路径（REST）或 GraphQL 操作名 |
| `description` | string | ❌ | API 描述 |
| `params` | array | ❌ | 请求参数定义 |
| `body` | object | ❌ | 请求体定义（POST/PUT/PATCH） |
| `response` | object | ✅ | 响应类型定义 |
| `auth` | boolean | ❌ | 是否需要认证（默认：false） |
| `cache` | object | ❌ | 缓存配置 |

### 参数定义

```yaml
params:
  - name: lat          # 参数名
    type: number       # 数据类型
    required: true     # 是否必填
    description: 纬度   # 描述
    default: 39.9      # 默认值
    min: -90           # 最小值（数字类型）
    max: 90            # 最大值（数字类型）
    pattern: "^\\d+$"  # 正则表达式（字符串类型）
    enum: [a, b, c]    # 枚举值
```

### 数据类型系统

支持以下基础类型：

| 类型 | Zod 映射 | 描述 |
|------|----------|------|
| `string` | `z.string()` | 字符串 |
| `number` | `z.number()` | 数字 |
| `boolean` | `z.boolean()` | 布尔值 |
| `integer` | `z.number().int()` | 整数 |
| `array` | `z.array()` | 数组 |
| `object` | `z.object()` | 对象 |
| `any` | `z.any()` | 任意类型 |
| `null` | `z.null()` | null 值 |

### 对象类型定义

```yaml
response:
  type: object
  properties:
    temperature:
      type: number
      description: 温度
    condition:
      type: string
      description: 天气状况
    tags:
      type: array
      items:
        type: string
  required:
    - temperature
    - condition
```

### 验证规则（Zod Schema）

支持以下验证规则：

| 规则 | 适用类型 | Zod 方法 |
|------|----------|----------|
| `required` | all | - |
| `default` | all | `.default()` |
| `min` | number/string | `.min()` |
| `max` | number/string | `.max()` |
| `pattern` | string | `.regex()` |
| `enum` | string/number | `.enum()` |
| `email` | string | `.email()` |
| `url` | string | `.url()` |
| `positive` | number | `.positive()` |
| `negative` | number | `.negative()` |

### 缓存配置

```yaml
cache:
  enabled: true
  ttl: 300           # 缓存时间（秒）
  key: "weather:{lat}:{lon}"  # 缓存键模板
```

## REST 接口配置示例

### 简单 GET 请求

```yaml
apis:
  - name: getWeather
    method: GET
    path: /api/weather
    description: 获取指定位置的天气信息
    params:
      - name: lat
        type: number
        required: true
        description: 纬度
        min: -90
        max: 90
      - name: lon
        type: number
        required: true
        description: 经度
        min: -180
        max: 180
    response:
      type: object
      properties:
        temperature:
          type: number
          description: 温度（摄氏度）
        condition:
          type: string
          description: 天气状况
        windSpeed:
          type: number
          description: 风速
      required:
        - temperature
        - condition
```

### POST 请求（带请求体）

```yaml
apis:
  - name: createUser
    method: POST
    path: /api/users
    description: 创建新用户
    body:
      type: object
      properties:
        username:
          type: string
          required: true
          min: 3
          max: 20
        email:
          type: string
          required: true
          email: true
        age:
          type: integer
          min: 0
          max: 150
      required:
        - username
        - email
    response:
      type: object
      properties:
        id:
          type: string
        username:
          type: string
        email:
          type: string
        createdAt:
          type: string
      required:
        - id
        - username
        - email
        - createdAt
```

### 带认证和缓存的接口

```yaml
apis:
  - name: getUserProfile
    method: GET
    path: /api/users/:id
    auth: true
    cache:
      enabled: true
      ttl: 600
      key: "user:profile:{id}"
    params:
      - name: id
        type: string
        required: true
        description: 用户 ID
    response:
      type: object
      properties:
        id:
          type: string
        username:
          type: string
        avatar:
          type: string
        bio:
          type: string
      required:
        - id
        - username
```

## GraphQL 接口配置示例

```yaml
apis:
  - name: getLaunches
    type: graphql
    path: launches
    description: 获取 SpaceX 火箭发射列表
    params:
      - name: limit
        type: integer
        default: 10
        description: 返回数量限制
      - name: offset
        type: integer
        default: 0
        description: 偏移量
    response:
      type: object
      properties:
        data:
          type: array
          items:
            type: object
            properties:
              id:
                type: string
              name:
                type: string
              dateUtc:
                type: string
              rocket:
                type: object
                properties:
                  id:
                    type: string
                  name:
                    type: string
            required:
              - id
              - name
              - dateUtc
      required:
        - data
```

## 自定义类型定义

可复用类型定义：

```yaml
types:
  PageInfo:
    type: object
    properties:
      total:
        type: integer
      page:
        type: integer
      pageSize:
        type: integer
      hasMore:
        type: boolean
    required:
      - total
      - page
      - pageSize
      - hasMore

  User:
    type: object
    properties:
      id:
        type: string
      username:
        type: string
      email:
        type: string
      avatar:
        type: string
    required:
      - id
      - username
      - email

apis:
  - name: listUsers
    path: /api/users
    params:
      - name: page
        type: integer
        default: 1
      - name: pageSize
        type: integer
        default: 20
    response:
      type: object
      properties:
        users:
          type: array
          items:
            $ref: User    # 引用自定义类型
        pagination:
          $ref: PageInfo  # 引用自定义类型
      required:
        - users
        - pagination
```

## 生成的代码结构

根据配置文件，自动生成以下代码：

```
src/
├── api/
│   ├── config.yaml          # 原始配置文件
│   ├── generated/
│   │   ├── routers.ts       # tRPC Router 定义
│   │   ├── schemas.ts       # Zod Schema 定义
│   │   └── types.ts         # TypeScript 类型定义
│   └── client.ts            # API 客户端入口
wundergraph/
├── wundergraph.config.ts    # WunderGraph 配置（自动更新）
└── operations/              # GraphQL Operations（自动生成）
```

### 生成的 Zod Schema 示例

```typescript
// src/api/generated/schemas.ts
import { z } from 'zod'

export const GetWeatherInputSchema = z.object({
  lat: z.number().min(-90).max(90),
  lon: z.number().min(-180).max(180),
})

export const GetWeatherResponseSchema = z.object({
  temperature: z.number(),
  condition: z.string(),
  windSpeed: z.number().optional(),
})
```

### 生成的 tRPC Router 示例

```typescript
// src/api/generated/routers.ts
import { router } from '../trpc'
import { GetWeatherInputSchema, GetWeatherResponseSchema } from './schemas'

export const weatherRouter = router({
  getWeather: {

```
