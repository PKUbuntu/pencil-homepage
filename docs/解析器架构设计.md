## 1. 概述

### 1.1 目标

设计一个模块化、可扩展的 .pen 文件解析器，将 Pencil 设计文件（JSON 格式）解析为抽象语法树（AST），再通过代码生成器将 AST 转换为目标代码（React/JSX + Tailwind CSS）。

### 1.2 设计原则

- **关注点分离**：解析逻辑、AST 表示、代码生成三者完全解耦
- **增量扩展**：新增元素类型无需修改核心框架，只需注册新的解析器和生成器
- **错误友好**：解析失败时提供清晰的错误定位和上下文信息
- **类型安全**：使用 TypeScript + Zod 进行运行时类型校验

### 2.2 模块职责

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **Loader** | 读取 .pen 文件，JSON 解析，Schema 校验 | 文件路径 | `PenDocument` |
| **VariableResolver** | 解析 `$--variable` 引用，构建变量映射表 | `PenDocument.variables` | `VariableMap` |
| **ThemeResolver** | 解析主题定义，处理多主题变体值 | `PenDocument.themes` + `VariableMap` | `ThemeConfig` |
| **ElementParser** | 递归解析元素树，分派到具体类型解析器 | `PenDocument.children` | `AST Node[]` |
| **ASTOptimizer** | 合并冗余节点、推断隐式属性、标准化 | `PenAST` | `PenAST`（优化后） |
| **CodeEmitter** | 将 AST 节点转换为目标代码片段 | `AST Node` | `CodeFragment` |
| **PostProcessor** | 格式化输出、去重样式、组合文件 | `CodeFragment[]` | 最终文件 |

## 3. 数据流

### 3.1 整体数据流

```
.pen 文件 (JSON)
    │
    │  ① Loader: 读取 + 校验
    ▼
PenDocument {
  version, themes, variables, children
}
    │
    │  ② VariableResolver: 解析变量
    │  ③ ThemeResolver: 解析主题
    ▼
ResolvedContext {
  variableMap: Map<string, ResolvedValue[]>,
  themeConfig: ThemeConfig
}
    │
    │  ④ ElementParser: 递归解析元素树
    ▼
PenAST {
  root: ASTNode,
  context: ResolvedContext
}
    │
    │  ⑤ ASTOptimizer: 优化 AST
    ▼
PenAST (optimized)
    │
    │  ⑥ CodeEmitter: 逐节点生成代码
    ▼
CodeFragment[] {
  jsx, styles, imports, exports
}
    │
    │  ⑦ PostProcessor: 组装输出
    ▼
OutputFiles {
  components/*.jsx,
  styles/variables.css,
  lib/theme.js
}
```

### 3.2 解析阶段详细流程

```
PenDocument.children (JSON Array)
    │
    ├── 遍历每个 child
    │   │
    │   ├── 读取 child.type
    │   │   │
    │   │   ├── "frame"  → FrameParser.parse(child)
    │   │   │                 ├── 解析布局属性 (layout, padding, gap, ...)
    │   │   │                 ├── 解析样式属性 (fill, stroke, cornerRadius, ...)
    │   │   │                 ├── 解析尺寸属性 (width, height)
    │   │   │                 └── 递归解析 child.children → ASTNode[]
    │   │   │
    │   │   ├── "text"   → TextParser.parse(child)
    │   │   │                 ├── 解析文本内容 (content)
    │   │   │                 ├── 解析排版属性 (fontSize, fontWeight, ...)
    │   │   │                 └── 解析样式属性 (fill, textAlign, lineHeight)
    │   │   │
    │   │   ├── "image"  → ImageParser.parse(child)  [预留]
    │   │   │
    │   │   └── unknown  → 记录警告，返回 UnknownNode
    │   │
    │   └── 返回 ASTNode
    │
    └── 返回 ASTNode[]
```

## 4. 核心接口定义

### 4.1 Loader 接口

```typescript
interface PenLoader {
  /**
   * 加载并校验 .pen 文件
   * @throws PenLoadError 当文件不存在或 JSON 格式错误时
   * @throws PenValidationError 当 Schema 校验失败时
   */
  load(filePath: string): Promise<PenDocument>;
}

/** .pen 文件顶层结构 */
interface PenDocument {
  version: string;
  themes: Record<string, string[]>;
  variables: Record<string, PenVariable>;
  children: PenElement[];
}

/** 变量定义 */
interface PenVariable {
  type: "color" | "number" | "string";
  value: PenVariableValue[];
}

interface PenVariableValue {
  value: string;
  theme?: Record<string, string>;  // 主题条件，如 { "Mode": "Dark" }
}
```

### 4.2 ElementParser 接口（插件式）

```typescript
/** 元素解析器接口 — 所有具体类型解析器必须实现 */
interface IElementParser<T extends ASTNode = ASTNode> {
  /** 该解析器处理的元素类型 */
  readonly type: string;
  
  /**
   * 解析单个元素
   * @param element 原始 .pen 元素 JSON
   * @param context 解析上下文（变量表、主题配置等）
   * @param parseChildren 递归解析子元素的回调
   * @returns 解析后的 AST 节点
   */
  parse(
    element: PenElement,
    context: ParseContext,
    parseChildren: (children: PenElement[]) => ASTNode[]
  ): T;
}

/** 解析器注册表 */
interface ParserRegistry {
  register(parser: IElementParser): void;
  getParser(type: string): IElementParser | undefined;
  getSupportedTypes(): string[];
}
```

### 4.3 CodeEmitter 接口（插件式）

```typescript
/** 代码发射器接口 — 所有具体节点生成器必须实现 */
interface ICodeEmitter<T extends ASTNode = ASTNode> {
  /** 该发射器处理的 AST 节点类型 */
  readonly nodeType: string;
  
  /**
   * 为单个 AST 节点生成代码片段
   * @param node AST 节点
   * @param context 生成上下文
   * @param emitChildren 递归生成子节点代码的回调
   * @returns 代码片段
   */
  emit(
    node: T,
    context: EmitContext,
    emitChildren: (children: ASTNode[]) => CodeFragment[]
  ): CodeFragment;
}

/** 代码片段 */
interface CodeFragment {
  jsx: string;           // JSX 代码
  styles?: string;       // 附加样式（如有）
  imports?: string[];    // 需要导入的模块
  exports?: string[];    // 需要导出的符号
}

/** 发射器注册表 */
interface EmitterRegistry {
  register(emitter: ICodeEmitter): void;
  getEmitter(nodeType: string): ICodeEmitter | undefined;
}
```

### 4.4 Pipeline 接口

```typescript
/** 解析器管线 — 串联所有阶段 */
interface PenPipeline {
  /**
   * 执行完整的解析-生成管线
   * @param input 输入 .pen 文件路径
   * @param options 管线选项
   * @returns 生成结果
   */
  run(input: string, options?: PipelineOptions): Promise<PipelineResult>;
}

interface PipelineOptions {
  /** 目标主题，默认 "Light" */
  theme?: string;
  /** 输出目录 */
  outDir?: string;
  /** 是否生成路由文件 */
  generateRouter?: boolean;
  /** 自定义元素解析器 */
  parsers?: IElementParser[];
  /** 自定义代码发射器 */
  emitters?: ICodeEmitter[];
}

interface PipelineResult {
  /** 生成的文件列表 */
  files: OutputFile[];
  /** 解析警告 */
  warnings: PenWarning[];
  /** AST（供调试） */
  ast: PenAST;
}
```

## 5. AST 结构定义

### 5.1 AST 节点类型层次

```
ASTNode (base)
├── FrameNode        → 容器/布局节点
│   ├── PageNode     → 页面级 Frame（顶层）
│   ├── SectionNode  → 语义化区域（Header, Footer, Hero 等）
│   └── ContainerNode → 通用容器
├── TextNode         → 文本节点
├── ImageNode        → 图片节点（预留）
├── ComponentRefNode → 组件引用节点（预留）
└── UnknownNode      → 未识别的节点类型
```

### 5.2 基础 AST 节点

```typescript
/** AST 节点基类 */
interface ASTNode {
  /** 节点类型标识 */
  type: string;
  /** 原始 .pen 元素 ID */
  id?: string;
  /** 元素名称 */
  name?: string;
  /** 样式属性 */
  styles: StyleProperties;
  /** 子节点 */
  children: ASTNode[];
  /** 源码位置信息（用于错误报告） */
  source: SourceLocation;
}

/** 源码位置 */
interface SourceLocation {
  /** .pen 文件路径 */
  file: string;
  /** 在 children 数组中的索引路径，如 [0, 2, 1] */
  path: number[];
  /** 原始 JSON 元素（用于错误报告） */
  raw: PenElement;
}
```

### 5.3 FrameNode（容器节点）

```typescript
interface FrameNode extends ASTNode {
  type: "frame";
  /** 语义角色（自动推断或从 name 映射） */
  semanticRole?: "page" | "header" | "footer" | "hero" | "section" | "card" | "button" | "nav" | "container";
  /** 布局属性 */
  layout: LayoutProperties;
}

interface LayoutProperties {
  /** 布局方向 */
  direction: "horizontal" | "vertical" | "none";
  /** 内边距 [top, right, bottom, left] 或统一值 */
  padding: number | [number, number] | [number, number, number, number];
  /** 子元素间距 */
  gap?: number;
  /** 主轴对齐 */
  justifyContent?: "start" | "center" | "end" | "space_between" | "space_around";
  /** 交叉轴对齐 */
  alignItems?: "start" | "center" | "end" | "stretch";
  /** 是否换行 */
  flexWrap?: boolean;
  /** 是否裁剪溢出 */
  clip?: boolean;
}
```

### 5.4 TextNode（文本节点）

```typescript
interface TextNode extends ASTNode {
  type: "text";
  /** 文本内容 */
  content: string;
  /** 排版属性 */
  typography: TypographyProperties;
}

interface TypographyProperties {
  fontFamily?: string;
  fontSize?: number;
  fontWeight?: string | number;
  lineHeight?: number;
  textAlign?: "left" | "center" | "right";
  textGrowth?: "auto" | "fixed-width";
}
```

### 5.5 通用样式属性

```typescript
interface StyleProperties {
  /** 尺寸 */
  width?: DimensionValue;
  height?: DimensionValue;
  /** 位置（仅 layout:"none" 时有效） */
  x?: number;
  y?: number;
  /** 填充色 */
  fill?: ColorValue;
  /** 边框 */
  stroke?: StrokeValue;
  /** 圆角 */
  cornerRadius?: number | [number, number, number, number];
  /** 不透明度 */
  opacity?: number;
}

/** 尺寸值：可以是数字、特殊关键字 */
type DimensionValue =
  | number                          // 固定像素值，如 1440
  | "fill_container"                // 填充父容器
  | "fit_content"                   // 自适应内容
  | `fit_content(${number})`;       // 自适应内容，带最小高度

/** 颜色值：可以是直接色值或变量引用 */
type ColorValue =
  | string                          // 直接色值，如 "#ffffff"
  | `$${string}`;                   // 变量引用，如 "$--background"

/** 边框值 */
interface StrokeValue {
  fill: ColorValue;
  thickness: number;
  position?: "inside" | "outside" | "center";
}
```

## 6. 错误处理机制

### 6.1 错误分类

| 错误级别 | 类型 | 说明 | 行为 |
|----------|------|------|------|
| **FATAL** | `PenLoadError` | 文件不存在、JSON 格式错误 | 终止解析，抛出异常 |
| **FATAL** | `PenValidationError` | Schema 校验失败（缺少 version/children） | 终止解析，抛出异常 |
| **ERROR** | `PenParseError` | 必需属性缺失（如 Frame 无 layout） | 记录错误，跳过该节点 |
| **WARN** | `PenWarning` | 未知元素类型、未识别的属性 | 记录警告，继续解析 |
| **INFO** | `PenInfo` | 隐式属性推断、默认值填充 | 记录信息，继续解析 |

### 6.2 错误结构

```typescript
interface PenError {
  /** 错误级别 */
  level: "fatal" | "error" | "warn" | "info";
  /** 错误代码 */
  code: string;
  /** 人类可读的错误信息 */
  message: string;
  /** 出错位置 */
  location: SourceLocation;
  /** 修复建议（可选） */
  suggestion?: string;
}
```

## 7. 扩展点设计

### 7.1 扩展点总览

解析器通过**注册表模式（Registry Pattern）**实现扩展，新增元素类型无需修改核心框架代码。

```
扩展点 1: ElementParser Registry
  → 注册新的元素类型解析器（如 ImageParser, SVGPathParser）

扩展点 2: CodeEmitter Registry
  → 注册新的代码生成策略（如 Vue 生成器、纯 HTML 生成器）

扩展点 3: AST Transform Plugin
  → 在 AST 优化阶段注入自定义转换（如语义化推断、组件抽取）

扩展点 4: PostProcessor Plugin
  → 在最终输出阶段注入自定义处理（如 Prettier 格式化、ESLint 修复）
```

### 7.2 新增元素类型的步骤

以新增 `image` 类型为例：

```typescript
// 1. 定义 AST 节点类型
interface ImageNode extends ASTNode {
  type: "image";
  src: string;
  alt?: string;
  objectFit?: "cover" | "contain" | "fill";
}

// 2. 实现 ElementParser
class ImageParser implements IElementParser<ImageNode> {
  readonly type = "image";
  
  parse(element: PenElement, context: ParseContext): ImageNode {
    return {
      type: "image",
      id: element.id,
      name: element.name,
      src: element.src ?? "",
      alt: element.alt ?? element.name ?? "",
      objectFit: element.objectFit ?? "cover",
      styles: this.parseStyles(element, context),
      children: [],
      source: context.currentLocation,
    };
  }
}

// 3. 实现 CodeEmitter
class ImageEmitter implements ICodeEmitter<ImageNode> {
  readonly nodeType = "image";
  
  emit(node: ImageNode): CodeFragment {
    return {
      jsx: `<img src="${node.src}" alt="${node.alt}" className="object-${node.objectFit}" />`,
    };
  }
}

// 4. 注册到管线
pipeline.parserRegistry.register(new ImageParser());
pipeline.emitterRegistry.register(new ImageEmitter());
```

### 7.3 扩展点：AST Transform

```typescript
/** AST 转换插件接口 */
interface IASTTransform {
  /** 插件名称 */
  readonly name: string;
  /** 执行顺序（数字越小越先执行） */
  readonly order: number;
  /** 转换 AST */
  transform(ast: PenAST): PenAST;
}

// 示例：语义角色推断插件
class SemanticRoleInferrer implements IASTTransform {
  readonly name = "semantic-role-inferrer";
  readonly order = 10;
  
  transform(ast: PenAST): PenAST {
    // 根据 name 属性推断 semanticRole
    // "Header" → header, "Footer" → footer, "Hero*" → hero, ...
    return this.walkAndInfer(ast);
  }
}
```

## 8. .pen 文件格式摘要

基于现有 .pen 文件分析，格式要点如下：

### 8.1 文件顶层结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | `string` | 格式版本，当前为 `"2.6"` |
| `themes` | `Record<string, string[]>` | 主题维度定义，如 `{ "Mode": ["Light", "Dark"] }` |
| `variables` | `Record<string, PenVariable>` | CSS 变量定义，支持多主题值 |
| `children` | `PenElement[]` | 顶层元素列表（通常为页面级 Frame） |

### 8.2 已识别的元素类型

| 类型 | 说明 | 出现频率 |
|------|------|----------|
| `frame` | 容器/布局元素，核心类型 | ★★★★★ |
| `text` | 文本元素 | ★★★★★ |
| `image` | 图片元素（预留） | ★☆☆☆☆ |
| `ref` | 组件引用（见 WORK_PLAN.md 模板） | ★☆☆☆☆ |

### 8.3 Frame 布局属性映射

| .pen 属性 | AST 属性 | Tailwind 映射 |
|-----------|----------|---------------|
| `layout: "horizontal"` | `direction: "horizontal"` | `flex flex-row` |
| `layout: "vertical"` | `direction: "vertical"` | `flex flex-col` |
| `layout: "none"` | `direction: "none"` | `relative` |
| `gap: N` | `gap: N` | `gap-[Npx]` |
| `padding: N` | `padding: N` | `p-[Npx]` |
| `padding: [V, H]` | `padding: [V, H, V, H]` | `py-[Vpx] px-[Hpx]` |
| `justifyContent: "space_between"` | `justifyContent: "space_between"` | `justify-between` |
| `alignItems: "center"` | `alignItems: "center"` | `items-center` |
| `flexWrap: "wrap"` | `flexWrap: true` | `flex-wrap` |

### 8.4 尺寸值映射

| .pen 值 | 含义 | CSS/Tailwind 映射 |
|---------|------|-------------------|
| `1440` | 固定 1440px | `w-[1440px]` |
| `"fill_container"` | 填满父容器 | `w-full` |
| `"fit_content"` | 自适应内容 | `w-fit` |
| `"fit_content(500)"` | 自适应，最小 500px | `min-h-[500px] h-fit` |
